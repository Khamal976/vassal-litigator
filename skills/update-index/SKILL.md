# Update-Index — Обновление и верификация индекса

Сканирует файловую систему дела и приводит `index.yaml` в соответствие с реальным состоянием файлов. Находит новые файлы без записи в индексе, «осиротевшие» записи без файлов, устаревшие зеркала.

## Предварительные условия

- Существует `.vassal/index.yaml`
- Существует `.vassal/case.yaml`

## Алгоритм

### Шаг 1 — Сканирование

1. Прочитай `.vassal/index.yaml`.
2. Просканируй файловую систему дела — все файлы в:
   - `Материалы от клиента/` (рекурсивно)
   - Все процессуальные папки `{ГГГГ-ММ-ДД} *` (рекурсивно)
   - Корень папки дела (кроме `.vassal/` и служебных файлов)
3. Игнорируй: `.vassal/` (внутренности), `На удаление/`, `Входящие документы/`, `.DS_Store`, `Thumbs.db`, `Таблица документов.xlsx`.

### Шаг 2 — Сравнение

4. Для каждого файла в файловой системе — проверь, есть ли запись в `index.yaml` (по полю `file`).
5. Для каждой записи в `index.yaml` — проверь, существует ли файл на диске.
6. Для каждого файла с записью — проверь, не устарело ли зеркало (файл изменился после `last_verified`).

Результат — три списка:
- **Новые файлы** (есть на диске, нет в индексе)
- **Осиротевшие записи** (есть в индексе, нет на диске)
- **Устаревшие зеркала** (файл изменился)

### Шаг 3 — Preview

7. Покажи Сюзерену результат сканирования:

```
Результат проверки индекса:

Файлов в индексе: 15
Файлов на диске: 17
Зеркал актуальных: 14

Новые файлы (нет в индексе):
  - Материалы от клиента/Иное/заметка.txt
  - 2026-04-01 Истец - пояснения/пояснения.pdf

Осиротевшие записи (файл не найден):
  - doc-005: Материалы от клиента/Переписка/старое_письмо.pdf

Устаревшие зеркала:
  - doc-003: mirrors/doc-003.md (файл изменён 2026-04-01)
```

8. Предложи действия:
   - Новые файлы → «Добавить в индекс и создать зеркала?»
   - Осиротевшие → «Удалить из индекса или файл был перемещён? Подскажи.»
   - Устаревшие → «Пересоздать зеркала?»

9. Дождись подтверждения. Если план пуст (0 новых, 0 осиротевших, 0 устаревших) — early-exit без записи в индекс и без backup, сообщи «Индекс синхронизирован с ФС, изменения не требуются».

### Шаг 3.5 — Snapshot (backup перед apply)

Перед любой записью в `.vassal/index.yaml` снимается резервная копия. Этот шаг **обязателен** при непустом плане изменений (см. также раздел «Постусловия» категории 4 в [shared/conventions.md](../../shared/conventions.md)).

10. Скопируй текущий `.vassal/index.yaml` в `.vassal/index.yaml.bak.{ГГГГ-ММ-ДД-ЧЧММСС}` (через `cp`, не перемещение).

11. Ротация: после создания нового бэкапа оставь **5 самых свежих** `.vassal/index.yaml.bak.*`. Остальные перемести в `На удаление/` (не `rm`, см. `shared/conventions.md` -- «Папка „На удаление/"»).

12. В `.vassal/history.md` зафиксируй имя созданного бэкапа в формате:
    ```
    Backup перед apply: .vassal/index.yaml.bak.{timestamp}
    Откат: cp .vassal/index.yaml.bak.{timestamp} .vassal/index.yaml
    ```
    Это нужно, чтобы Сюзерен мог откатиться вручную, если post-apply верификация провалится или он передумает.

### Шаг 4 — Apply

13. **Новые файлы**: для каждого —
    - Присвой ID (`doc-{next_id}`)
    - Извлеки текст (OCR pipeline)
    - Вычисли `content_hash` (SHA-256 по бинарному содержимому, см. `shared/conventions.md` → «Дедупликация по `content_hash`»)
    - Создай md-зеркало
    - Добавь запись в index.yaml
    - Определи тип, дату, стороны (делегируй Haiku при необходимости)

14. **Осиротевшие записи**: по указанию Сюзерена —
    - Удалить запись из индекса
    - Или обновить путь, если файл перемещён
    - Если запись удалена — переместить md-зеркало в `На удаление/` (не `rm`, см. `shared/conventions.md`)

15. **Устаревшие зеркала**: пересоздай md-зеркало (делегируй Haiku-субагенту), обнови `content_hash` и `last_verified`.

16. Обнови `last_updated` (top-level) и `last_verified` для **всех** прошедших проверку записей (не только изменённых) в `index.yaml`.
17. Добавь запись в `.vassal/history.md` с итогом: добавлено/удалено/пересоздано/имя бэкапа.

### Шаг 4.5 — Rollback при сбое apply

Если apply упал посередине (например, при добавлении 8-й из 15 записей), `index.yaml` остался в промежуточном состоянии:

18. Восстанови `.vassal/index.yaml` из последнего бэкапа: `cp .vassal/index.yaml.bak.{timestamp} .vassal/index.yaml`.
19. Зеркала, успевшие создаться частично — перемести в `На удаление/` (по списку из шага 13).
20. Сообщи Сюзерену: «Apply сломался на записи K из N. Индекс восстановлен из бэкапа `{timestamp}`. Рекомендую: разобрать причину сбоя и перезапустить.»

### Шаг 5 — Постусловия (верификация после apply)

Полный чек-лист — см. раздел «Постусловия скиллов» / «Категория 4: служебные» в [shared/conventions.md](../../shared/conventions.md). Кратко:

21. **Целостность индекса:**
    - `index.yaml` — валидный YAML, соответствует [shared/index-schema.yaml](../../shared/index-schema.yaml).
    - Нет дубликатов `id` и `file`.
    - `next_id` > max(существующих ID).

22. **Двусторонняя связь «запись ↔ зеркало»:**
    - Каждая запись в индексе имеет зеркало в `.vassal/mirrors/{id}.md`.
    - **Каждое зеркало в `.vassal/mirrors/` имеет запись в индексе** (orphan mirrors → классифицировать как «зеркало без записи», предложить Сюзерену удалить или восстановить запись).
    - Каждое зеркало имеет валидный frontmatter (`id`, `source`, `type`).

23. **Backup существует:** `.vassal/index.yaml.bak.{timestamp}` создан до apply (см. шаг 10).

24. **Если найдены проблемы** — предложить откат: `cp .vassal/index.yaml.bak.{timestamp} .vassal/index.yaml`.

## Идемпотентность

Повторный запуск безопасен:
- Если план изменений пуст (0 новых, 0 осиротевших, 0 устаревших) — early-exit, бэкап не создаётся, `last_updated` не меняется (см. шаг 9).
- Содержание уже синхронизированных файлов не пересоздаётся (`content_hash` сверяется до решения о пересоздании зеркала, см. шаг 13).

## Граничные случаи

- **Файл переименован вручную** → старая запись «осиротеет», новое имя -- «новый файл». Если `content_hash` совпадает -- автоматически связать (обновить `file:` в существующей записи), без вопроса Сюзерену (см. `shared/conventions.md` → «Дедупликация по `content_hash`»). Если хэш не совпадает -- предложить связать вручную или подтвердить как разные документы.
- **Пустой индекс при существующих файлах на диске** -- это массовое «новые файлы без записи». Не запускай intake автоматически: предложи Сюзерену выбрать -- запустить intake (если файлы только пришли от клиента) или обработать через update-index (если структура папок дела перенесена с другого диска без `.vassal/`).
- **Пустой индекс при пустом диске** -- предложи запустить intake.
- **Файл в `.vassal/raw/` появился без обработки** -- это нормально, raw/ не индексируется.
- **Запись с `filing_status: submitted | filed`, файл не найден** -- **не предлагать удаление**. Сообщи Сюзерену: «документ подан в суд, но файл в локальной ФС не найден -- восстановить из `.vassal/raw/` или обновить путь?». Поданный документ юридически существует в материалах суда, удалять запись нельзя.
- **Дубликат `file:` в двух записях** -- остановиться, показать обе записи Сюзерену, дождаться решения (какую оставить, какую слить). Не разрешать автоматически.
