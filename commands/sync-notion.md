# sync-notion -- Синхронизация с Notion

Когда пользователь запускает эту команду с "$ARGUMENTS":

1. Прочитай скилл `skills/notion-sync/SKILL.md` и следуй его инструкциям.
2. Если `$ARGUMENTS` содержит `--dry-run` -- выполни только Phase 0-1 (загрузка конфига и чтение локальных источников), покажи Сюзерену, что **будет** записано в Notion, но фактический upsert не выполняй.
3. Если `$ARGUMENTS` содержит `--force` -- проигнорируй `fields_manual_only` из конфига и перезапиши все поля по локальной правде. **Используй с осторожностью** -- сбросит ручные правки Сюзерена в Notion.
4. По умолчанию (без аргументов) -- стандартный pipeline: load config → read local → upsert Cases → upsert Judges → log.
5. Убедись, что `.vassal/case.yaml` существует (иначе нечего синкать).
6. Убедись, что `~/.vassal/notion-config.yaml` существует (иначе предупреди и предложи bootstrap по `scripts/notion-init.md`).
7. После apply покажи Сюзерену короткую сводку: что создано/обновлено в Notion, ссылки на страницы (если MCP вернул URL).
