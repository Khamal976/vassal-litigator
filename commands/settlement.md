# settlement -- Мировое соглашение / отказ от иска / признание иска

Когда пользователь запускает эту команду с "$ARGUMENTS":

1. Прочитай скилл `skills/settlement/SKILL.md` и следуй его инструкциям.
2. Если `$ARGUMENTS` содержит данные о режиме (мировое / отказ от иска / признание иска), стадии (1 инстанция / апелляция / кассация / исполнительное), инициаторе (мы / оппонент / совместно), пути к проекту от оппонента, особых указаниях -- извлеки из них и используй как стартовые ответы на фазу «Входные данные» (не переспрашивай уже данное).
3. Если аргументов нет или данные неполные -- собери недостающее через 5 уточняющих вопросов (см. SKILL.md → «Входные данные от Сюзерена»: mode, stage, initiator, дата ближайшего заседания, особые указания).
4. Выполни полный pipeline из SKILL.md: фаза 0 pre-flight (включая проверку in_progress / confirmed_date / совместимости mode×stage / сигнала банкротство) -> фаза 1 загрузка контекста -> фаза 2 анализ зоны компромисса (Opus) -> фаза 3 стресс-тест 3 рисков (Opus) -> фаза 4 подготовка документов (ветвление по mode и по stage) -> фаза 5 preview -> фаза 6 apply -> фаза 7 сводка для Сюзерена.
