# backfill-global -- Бэкфилл глобальной памяти

Когда пользователь запускает эту команду:

1. Прочитай скилл `skills/backfill-global/SKILL.md` и следуй его инструкциям.
2. **Парсинг аргументов:**
   - Позиционные аргументы -- путь(и) к корням дел (одно дело, папка-родитель для рекурсивного поиска, или список).
   - Флаг `--no-merge` -- выключить Opus-этап семантической дедупликации (см. SKILL.md → «Опции команды»).
   - Если позиционных аргументов нет -- спроси Сюзерена, где лежат корни дел.
3. Выполни полный pipeline: сбор дел → сбор источников → резолв целевых глобальных файлов → Opus-маппинг с семантическим дедупом (или пропуск при `--no-merge`) → preview → apply → опциональный sync-notion.
4. Перед apply -- обязательно дождись подтверждения Сюзерена. Принимаемые ответы: `yes` / `apply`, `отклонить merge {номера}`, `выборочно`, `нет`. См. SKILL.md → фаза 4.

Примеры запуска:
- `/vassal-litigator:backfill-global C:\Cases\А33-12345-2026` -- одно дело
- `/vassal-litigator:backfill-global C:\Cases\` -- рекурсивный поиск по всем подпапкам
- `/vassal-litigator:backfill-global C:\Cases\ --no-merge` -- то же, но без семантической дедупликации
