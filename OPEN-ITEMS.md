### A.3. Backfill глобальной памяти для существующих дел ⏳ **Средний приоритет, разово -- без срочности**

**Что сейчас:** глобальная память (`$VASSAL_GLOBAL_DIR/`) органически наполняется только при следующих запусках `analyze-hearing` / `draft-judgment` / `add-opponent` в делах. Уже накопленные в `.vassal/hearings/judge-profile.md` и `.vassal/analysis/opponent-*.md` наблюдения по существующим делам **не переносятся автоматически**.

**Когда выстреливает:** если есть дела с 3+ заседаниями у одного судьи в локальной памяти, но в глобальной -- пусто. Значимость возрастает с длительностью практики.

**Ответ Сюзерена (2026-05-11):** дел с 3+ заседаниями у одного судьи пока немного, но **точно появятся**. Пункт остаётся в плане, **без срочности** -- запускать после накопления критической массы (3+ заседания у нескольких судей в разных делах), чтобы бэкфилл дал ощутимый эффект.

**План:** этап 6.3 -- новый утилитарный скилл `vassal-backfill-global`. Алгоритм: «дай список путей к корням дел (или папку-родитель) → для каждого читаю `.vassal/hearings/judge-profile.md` + `opponent-*.md` → аппенжу в глобальные файлы с маркерами `(дело {номер}, заседание/документ {дата})` → отчитываюсь о добавленном». Цена -- 30-45 мин на скилл, плюс время на сам запуск (зависит от числа дел).

### A.5. OneDrive conflict copies при параллельной работе ❌ **Принято как ограничение**

**Что сейчас:** если на двух машинах одновременно запустить скиллы, пишущие в один и тот же глобальный файл (например, `judges/Иванов-И-И--АС-N.md`), OneDrive создаст `... (kholv's conflicted copy ...).md`.

**Что задокументировано:** этап 6.1, раздел ограничений -- «не работать параллельно, дать OneDrive 1-2 мин на синк при переключении».

**Возможные fix'ы (отвергнуты):**
- Lock-файл соседом -- race-condition-prone, может застрять.
- Централизованный lock-сервер -- overkill для однопользовательского сценария.

**Решение:** оставить как есть, документация уже есть. Если столкнёшься с реальным конфликтом, разрешать мёрдж руками (аппенд-only логика этому помогает -- ничего не перезаписано, просто две параллельные ветки наблюдений).
